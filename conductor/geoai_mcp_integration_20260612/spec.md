# Specification: GeoAI MCP Integration

## Overview
This track integrates the pre-existing GeoAI MCP Server into the FarmVibes.AI platform to provide high-accuracy, zero-shot interactive segmentation for agriculture and sustainability use cases, particularly rooftop segmentation.

## Functional Requirements
- **Frontend Integration:** The client-side (geoai.js) will be refactored to consume the GeoAI MCP Server via the Model Context Protocol, replacing local mock implementations.
- **Backend Workflow Integration:** FarmVibes.AI backend workflows will be updated to orchestrate data flow between the MCP Server's output directory and the FarmVibes.AI data infrastructure.
- **Data Persistence:** Segmentation results (masks/polygons) generated by the MCP Server will be automatically ingested, curated, and stored in the FarmVibes.AI `50_curated` and `70_artifacts` directories.

## Non-Functional Requirements
- **Performance:** Pipeline is prioritized for **High Accuracy** as requested.
- **Maintainability:** Standardized on MCP for all geospatial tool interactions.

## Acceptance Criteria
- [ ] Frontend successfully triggers segmentation tools on the GeoAI MCP Server.
- [ ] MCP Server successfully processes input imagery and saves results to the shared volume/directory.
- [ ] Backend workflow detects new outputs, curates them, and persists them into the platform structure.
- [ ] Pipeline produces verifiable high-accuracy rooftop segmentations.

## Out of Scope
- Direct model training or fine-tuning.
- Real-time video stream processing.
